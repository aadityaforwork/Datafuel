A FastAPI service that reads a real Amazon Ads CSV report, cleans the data,
computes per-campaign performance metrics, labels each campaign, and exposes
account-level summary and actionable insights.

## Project Structure

```
datafuel-assessment/
├── main.py                 # FastAPI routes
├── services.py             # Business logic (analyze, summary, insights, recommendations)
├── utils.py                # Parsing + safe-math helpers (bugs fixed here)
├── models.py               # Pydantic response models
├── data/
│   ├── amazon_ads_data.csv       # Source CSV (provided)
│   └── campaigns_analyzed.csv    # Generated by /analyze
├── requirements.txt
└── README.md
```

## Run

```bash
# 1. Install deps (venv already included in this repo)
venv/Scripts/python.exe -m pip install -r requirements.txt

# 2. Start the API
venv/Scripts/uvicorn.exe main:app --reload

# 3. Hit the endpoints
curl http://127.0.0.1:8000/analyze
curl http://127.0.0.1:8000/summary
curl http://127.0.0.1:8000/insights
curl http://127.0.0.1:8000/recommendations

# Or open interactive docs
# http://127.0.0.1:8000/docs
```

## Step 0 — Bug Fixes in `utils.py`

| # | Function | Bug | Fix |
|---|---|---|---|
| 1 | `parse_budget` | Crashes on `None`, empty string, or `"₹"`-only input (`None.replace` / `float("")`) | Guard `None`; regex-strip all non-numeric chars; return `0.0` if nothing is left |
| 2 | `compute_acos` | `ZeroDivisionError` when `sales == 0`; also was missing `× 100` per spec table | Return `0.0` if sales is falsy; multiply by 100 to match the spec's ACOS formula |
| 3 | `compute_ctr` | Crash when `impressions == 0`; also was returning an unrounded float while other metrics round | Guard zero impressions; round to 4 decimals for consistency |

## Metric Formulas

All formulas match the spec exactly; each one returns `0` on its documented edge case instead of raising or returning `inf`/`NaN`.

| Metric | Formula | Zero Case |
|---|---|---|
| CTR | `clicks / impressions × 100` | impressions = 0 → 0 |
| CPC | `spend / clicks` | clicks = 0 → 0 |
| Conversion Rate | `orders / clicks × 100` | clicks = 0 → 0 |
| ROAS | `sales / spend` | spend = 0 → 0 |
| ACOS | `spend / sales × 100` | sales = 0 → 0 |

## Performance Labels

| Label | Rule | Meaning |
|---|---|---|
| Scale | ROAS > 3 | Profitable, increase budget |
| Optimize | 1 ≤ ROAS ≤ 3 | Borderline, improve targeting |
| Pause | ROAS < 1 | Losing money, stop spending |

## Endpoints

### `GET /analyze`
Reads `data/amazon_ads_data.csv`, cleans every row, computes all five metrics,
labels each campaign, writes the enriched data to `data/campaigns_analyzed.csv`,
and returns the full list as JSON.

Malformed rows (missing campaign name, negative values, un-parseable numbers)
are skipped and logged — the endpoint never crashes on bad data.

### `GET /summary`
Account-level roll-up:
- `total_spend`, `total_sales`, `overall_roas`
- `best_campaign` / `worst_campaign` (by ROAS, considering only campaigns with real spend so paused-from-birth rows don't dominate "worst")
- `label_breakdown` — count per label
- `wasted_spend_pct` — share of total spend going to Pause-labelled campaigns

### `GET /insights`
Flags any campaign that trips any of these rules and attaches the highest-priority recommendation:

| Priority | Trigger | Recommendation | Why |
|---|---|---|---|
| 1 | Spend > 0 AND Orders = 0 | **Pause** | Pure budget drain — no ROI at all |
| 2 | Budget > 0, Impressions = 0, Spend = 0 | **Check Targeting** | Budget is live but ads aren't serving — bids or targeting too restrictive |
| 3 | ACOS > 80% with Sales > 0 | **Reduce Budget** | Inefficient but converting — tighten bids/keywords |
| 4 | Impressions > 0 AND CTR < 0.3% | **Review Creative** | Ads are showing but not being clicked — creative/relevance problem |

Each flagged campaign returns the original metric value that triggered it, plus a short reason string.

### `GET /recommendations` (Bonus)
A ranked list of the **top 3 actions** the seller should take today, sorted by estimated rupee impact.
Pulls candidates from three buckets:
1. **Pause** biggest zero-conversion spenders → savings realized immediately
2. **Increase Budget** on Scale-labelled top-ROAS campaigns → projected +50% sales at 50% more budget
3. **Reduce Budget** on high-ACOS converters → recoverable spend = `spend − sales × 0.8` (bring ACOS to the 80% threshold)

**Why this endpoint:** `/insights` tells you what's broken; `/recommendations` tells you what to do *first*. An Amazon seller looking at 348 campaigns doesn't want a list of 92 problems — they want the 3 moves that matter today.

## Design Notes

- **Separation of concerns:** FastAPI routes in `main.py` do nothing but delegate. All business logic lives in `services.py`. Parsing primitives live in `utils.py`. Response shapes in `models.py`.
- **CSV encoding:** opened with `utf-8-sig` to strip any BOM; `parse_budget` regex-strips every non-numeric character so it tolerates both `₹` and any mojibake form (`â¹`) that might appear from a mis-saved source file.
- **Safe math everywhere:** every division goes through helpers that short-circuit to `0.0` on a zero denominator, so Infinity/NaN never reach the response.
- **Best/worst filter:** the "worst" campaign is picked from those with actual spend so the summary isn't dominated by campaigns that never ran.
